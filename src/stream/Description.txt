Ленивая потоковая обработка данных


int res = Stream(1, 2, 3, 4, 5) 			// создаём последовательность от 1 до 5
    | map([](int a) { return a * a;  }) // применяем лямбду-функцию возведения в квадрат для каждого элемента
    | sum();							// суммируем все элементы (инициализируем сумму нулём на всякий случай, 
                                        // вдруг поток данных пуст)


Конечно, мы можем записать это и таким образом:

for (int i = 1; i <= 5; i++) {
    res2 += i * i;
}

И второй кусок кода выполнится быстрее (потому что компилятору проще оптимизировать именно его). 
В чём же тогда преимущество?

- Преимущество заключается в том, что нам не приходится напрягаться и думать:
    а. о границах счётчика цикла
    б. о самом цикле
    в. вместо описания тела цикла мы поставляем только конкретный оператор, 
        который хотим применить к элементам потока

Т.е. мы можем на время забыть про конкретную реализацию данного куска алгоритма 
и сосредоточиться на идейной реализации алгоритма или на решении самой задачи.
